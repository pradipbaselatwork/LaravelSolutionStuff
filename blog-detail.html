In this tutorial, we will implement ACL (Access Control List) in Laravel 12 using roles and permissions. We'll use the spatie/laravel-permission package to manage roles, permissions, and assign them to users.

Laravel 12 ACL Coding Part

Below are the coding steps after installing Laravel and setting up roles and permissions:

Install Package & Configure


composer require spatie/laravel-permission

php artisan vendor:publish --provider="Spatie\Permission\PermissionServiceProvider"
php artisan migrate

Create Models & Relationships


// User.php
use Spatie\Permission\Traits\HasRoles;

class User extends Authenticatable {
    use HasRoles;
}

Add Middleware to Routes


// web.php
Route::group(['middleware' => ['role:admin']], function () {
    Route::get('/admin', [AdminController::class, 'index']);
});

Create Seeder for Roles & Permissions


// DatabaseSeeder.php
use Spatie\Permission\Models\Role;
use Spatie\Permission\Models\Permission;

Role::create(['name' => 'admin']);
Permission::create(['name' => 'edit products']);

Create Controllers & Views


// ProductController.php
public function index() {
    $products = Product::all();
    return view('products.index', compact('products'));
}

Follow these steps carefully and you will have a fully functional ACL system in Laravel 12 with roles and permissions.
